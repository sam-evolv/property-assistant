## 1. Audit Logging for Message/Chat Viewing

When a developer views individual purchaser messages or clicks into a specific unit's chat history, log it.

### Database

```sql
-- Create audit log table if not exists
CREATE TABLE IF NOT EXISTS data_access_log (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  accessor_id UUID NOT NULL,
  accessor_email TEXT,
  accessor_role TEXT,
  action TEXT NOT NULL,
  resource_type TEXT NOT NULL,
  resource_id UUID,
  resource_description TEXT,
  ip_address TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Index for querying
CREATE INDEX idx_data_access_log_created ON data_access_log(created_at);
CREATE INDEX idx_data_access_log_accessor ON data_access_log(accessor_id);
```

### What to Log

Log whenever a developer/admin:
- Views the messages/chat analytics page for a development
- Views individual message details
- Views a specific unit's chat history
- Exports any data containing personal information

### Log Entry Example

```javascript
{
  accessor_id: "developer-uuid",
  accessor_email: "sam@evolvai.ie",
  accessor_role: "developer",
  action: "viewed_chat_history",
  resource_type: "unit_messages",
  resource_id: "unit-uuid",
  resource_description: "Viewed chat history for 15 Ãrdan View",
  ip_address: "192.168.1.1",
  created_at: "2026-02-01T12:00:00Z"
}
```

### Implementation

Add logging call in these locations:
- Chat Analytics page load
- Individual message view
- Unit detail page (if showing messages)
- Any data export function

```javascript
async function logDataAccess(userId, userEmail, role, action, resourceType, resourceId, description) {
  await supabase.from('data_access_log').insert({
    accessor_id: userId,

    accessor_role: role,
    action: action,
    resource_type: resourceType,
    resource_id: resourceId,
    resource_description: description
  });
}
```
