# Session 5A Diagnosis — Drafts pipeline regressions

Findings after reading the live code on `main`:

1. **Only one chat route exists**: `app/api/agent-intelligence/chat/route.ts`. There is no `app/api/agent/intelligence/chat/route.ts`, and both Intelligence pages (mobile `app/agent/intelligence/page.tsx` and desktop `app/agent/dashboard/intelligence/page.tsx`) POST to `/api/agent-intelligence/chat`. No dual-route reconciliation to do; the route under `app/api/agent/intelligence/*` is the drafts/send/autonomy/voice-action infra, not a chat route.
2. **Chat route never emits envelopes**. The SSE loop only emits `tools_used`, `token`, `followups`, `done`, `error`. The `AgenticSkillEnvelope` returned by `runAgenticSkill()` (via `ToolResult.data`) is stringified back to the model as a tool message and dropped on the client. No `envelope` frame is sent over the wire.
3. **Agentic skills do NOT write to `pending_drafts`**. `chaseAgedContracts`, `draftViewingFollowup`, `weeklyMondayBriefing`, `draftLeaseRenewal`, `naturalQuery`, `scheduleViewingDraft` all generate `randomUUID()` in-memory draft ids and return them in the envelope. Nothing lands in `pending_drafts`. The drafts inbox is therefore blind to anything Intelligence chat produces.
4. **No ApprovalDrawer, drawer store, envelope types, or provider exist in the repo**. `grep` for `ApprovalDrawer`, `drawer-store`, `openApprovalDrawer`, `ApprovalDrawerProvider` returns zero hits. The historical drawer shell from `claude/approval-drawer-shell-lavwT` is not on `main`. This session must build it.
5. **System prompt invites narration**. `lib/agent-intelligence/system-prompt.ts` says "Summarise each draft: who it's to, subject line, why it was generated." Combined with the `chaseAgedContracts` envelope containing placeholder `solicitor@tbc.invalid` recipients and the fact that envelopes never reach the client, the model has no incentive to do anything but list names. No explicit "call the tool, don't narrate" rule, no worked examples.
6. **Voice capture flow is unrelated to the chat drawer** and already writes to `pending_drafts` via `app/api/agent/intelligence/execute-actions/route.ts`. Session 1 `VoiceConfirmationCard` remains the voice review UX and is not touched by this session.

Fix plan: introduce `lib/agent-intelligence/draft-store.ts` with `persistDraftsForEnvelope()`, thread every agentic skill through it so each in-envelope draft has a real `pending_drafts.id`, emit `envelope` SSE frames from the chat route, harden the system prompt, build the drawer store + provider + `ApprovalDrawer` UI, wire it into both Intelligence pages.
